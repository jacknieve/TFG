1 Objetivos

OBJ-0001 Gestionar el mentoring
Versión 1.0 ( 19/02/2022 )
Autores ?
Fuentes ?
Descripción El sistema deberá permitir la gestión del mentoring. Los [ACT-0002] accederán a la aplicación para poder ponerse en contacto con los [ACT-0001] para que estos los aconsejen, apoyen o guíen en algún aspecto de su carrera profesional.
Subobjetivos
  • [OBJ-0003] Gestión de usuarios: El sistema deberá permitir a los [ACT-0001] y a los [ACT-0002] registrarse en la aplicación, creandose un perfil con sus datos personales, profesionales y de contacto. Los usuarios podrán modificar su propio perfil y acceder a los perfiles de otros usuarios, mediante una búsqueda.
  • [OBJ-0004] Contacto entre usuarios: El sistema deberá permitir que los [ACT-0002] tengan la posibilidad de ponerse en contacto con los [ACT-0001].
  • [OBJ-0005] Compartir ficheros: El sistema deberá permitir a los ususarios compartir ficheros con otros usuarios, ya sea exponiendolos en su perfil o enviándoselos a otro usuario con quien hayan establecido un contacto previo.
  • [OBJ-0006] Generación de reportes: El sistema deberá permitir generar reportes a partir de los datos de la aplicación, para que los usuarios puedan concocer el estado y evolución de la misma.
  • [OBJ-0008] Modular: El sistema deberá ser modular para permitir realizar modificaciones en uno de sus módulos o añadir nuevos módulos sin afectar de manera importante a los demás módulos.

    4 Requisitos funcionales y no funcionales

    Los requisitos funcionales indican las funcionalidades que debe proveer la aplicación, mientras que los no funcionales, indican restricciones que se deben cumplir.
    Los requisitos que tienen un * implican funcionalidades extra que no estarían abarcadas dentro del alcance del proyecto, y que es posible que no figuren dentro del análisis de requisitos final. En cambio, si pueden formar parte del aparatado de futuro del proyecto.
    Los requisitos que tienen comentario pueden verse alterados por las circunstancias que recogen dichos comentarios.

  • [CRQ-0002] Mentorizado
    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un usuario quiera proceder a registrarse en la aplicación.
    Precondición El usuario no puede haber iniciado sesión.
    Secuencia normal Paso Acción
    1 El actor Usuario (ACT-0007) escoge con que rol quiere registrarse (mentor o mentorizado).
    2 El actor Usuario (ACT-0007) introduce los datos solicitados por la aplicación para su registro, y acepta los términos y condiciones.
    3 El sistema guarda los datos del usuario en el repositorio, marcando la cuenta como no verificada, y envía un correo electrónico a la dirección de correo facilitada con un enlace para verificar la cuenta.
    4 El actor Usuario (ACT-0007) accede a su correo electrónico y entra en el enlace para verificar su cuenta.
    5 El sistema debe actualizar el repositorio para indicar que la cuenta se ha verificado e indicar al usuario que la operación ha sido un éxito y que ya puede proceder a inciar sesión.
    Postcondición El usuario debe quedar registrado en el repositorio de datos y se le debe permirtir iniciar sesión en la aplicación.
    Excepciones Paso Acción
    2 Si el usuario cambia de idea y le da a cancelar, el sistema redirije al usuario a la página de inicio, a continuación este caso de uso queda sin efecto
    2 Si el usuario no indica que acepta los términos y condiciones al aceptar, el sistema le indica al usuario que debe aceptarlos si quiere registrarse, a continuación este caso de uso continúa ( regresa al paso 2)
    2 Si el usuario no cubre uno de los campos obligatorios en el registro y le da a aceptar, el sistema le indica al usuario que debe rellenar esos campos si se quiere registrar, a continuación este caso de uso continúa ( regresa al paso 2)
    2 Si el usuario cubre uno de los campos con datos no validos, el sistema le indica al usuario que datos no son validos y como deberían ser, a continuación este caso de uso continúa ( regresa al paso 2)
    3 Si el sistema detecta que ya hay una cuenta registrada con el correo electrónico facilitado, el sistema le indica al usuario que ya existe una cuenta con ese correo electrónico, y le pide que inicie sesión o utilice otro correo, a continuación este caso de uso continúa ( regresa al paso 2)
    3 Si se produce un error al acceder al repositorio de datos, el sistema se lo notificará al usuario, indicandole que vuelva a intentarlo más tarde, a continuación este caso de uso continúa ( regresa al paso 2)
    4 Si el usuario no verifica su cuenta pasadas 48 horas, el sistema envía una correo electrónico a la dirreción de correo indicando que el tiempo ha expirado y elimina los datos registrados de ese usuario, a continuación este caso de uso queda sin efecto
    5 Si se produce un error al acceder al repositorio de datos, el sistema se lo notificará al usuario mediante un correo electrónico para que lo vuelva a intentar más tarde, a continuación este caso de uso continúa ( regresa al paso 4)
